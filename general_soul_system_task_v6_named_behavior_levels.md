# 武魂系统任务书 v6（行为阶名版）

## 一、核心目标

在现有双等级系统基础上，新增 **行为阶名**。

系统中现在有三套概念，必须分清：

1. **战评等级**：神将 / 飞将 / 名将 / 骁将 / 健将 / 勇将 / 战将  
   - 反映：这一整局打得有多好
   - 来源：总战功评价

2. **行为等级（序列等级）**：Lv7 ~ Lv1  
   - 反映：玩家在主行为线上打到什么段位
   - 来源：主行为得分
   - 用途：程序查表

3. **行为阶名**：每条行为线对应一套独立的 7 阶名字  
   - 反映：这条行为路线的风格化层级称号
   - 用途：前端展示
   - 例如：疾袭先登 · 裂阵 / 不伤而胜 · 轻裁 / 中军主宰 · 元戎

---

## 二、最终展示逻辑

固定为：

**战评等级 + 主行为 + 行为阶名 + 武魂**

例如：

- 战评：飞将
- 主行为：疾袭先登
- 行为阶名：裂阵
- 武魂：张辽

前端推荐写法：

> **战评：飞将**  
> **主行为：疾袭先登 · 裂阵**  
> **武魂：张辽**

或者：

> **你此战评定为飞将。**  
> **主行疾袭先登，其势已至「裂阵」。**  
> **故映得武魂：张辽。**

---

## 三、行为阶名规则

行为阶名不是单独计算出来的。  
它只是对 `(行为, 序列等级)` 的**文案翻译层**。

也就是说：

- 程序内部仍然用：`行为 + 序列等级`
- 前端展示时，把这个组合翻译成：`行为 + 行为阶名`

取得分最高的行为，作为主行为。

### 3. 序列等级判定
根据主行为得分，映射出 Lv7 ~ Lv1。

### 4. 查表
使用 `(行为, 序列等级)` 在 CSV 中查表。

### 5. 返回结果
结果中必须包含：

- 行为
- 序列等级
- 行为阶名
- 武将
- 综合等级
- 武将详细信息
